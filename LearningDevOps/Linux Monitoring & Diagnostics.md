## Краткое объяснение
Прежде чем мы уйдем от Linux к сетям, нужно научиться «слушать» сервер. В DevOps важно не только запустить сервис, но и понять, почему он тормозит или почему закончилось место на диске. Для этого используются утилиты мониторинга ресурсов (CPU, RAM, Disk I/O) в реальном времени.

> [!note] Определение
> **Мониторинг** — процесс наблюдения за состоянием сервера (CPU, RAM, Disk) для предотвращения аварий и оптимизации ресурсов [2].

## Ключевые понятия темы

- **Load Average**: средняя нагрузка на систему за 1, 5 и 15 минут.
- **htop / top**: интерактивные диспетчеры задач.
- **df / du**: проверка свободного места на дисках и размера папок.
- **free**: информация об использовании оперативной памяти.
- **iostat / netstat**: статистика дискового ввода-вывода и сетевых соединений
  
## Практика

Для начала установим `htop` и запустим его:
```terminal
sudo apt update && sudo apt install htop -y
```

После установки вызваем `htop`:
```terminal
htop
```

Вот что мы увидим при вызове диспетчера задач(# Посмотри на загрузку ядер и потребление памяти. Нажми F10 для выхода.):
![[Pasted image 20260309202034.png]]

Далее используем полезную команду для проверки увесестых папок:
```terminal
sudo du -sh /var/log/* | sort -rh | head -n 5 
```
Разберем команду для понимания:

- За что отвечает `sudo`?
	     Sudo отвечает за выполнение команд от имени `root`. Это важно потому что ы ином случае ты не сможешь прочитать логи, т.к. `/var/log` пренадлежит `root`
- За что отвечает `du -sh /var/log/*`?
		`du` отвечает за показ раз
