
Все логи находятся по пути `var/logs`

В нашем случае мы смотрим логи Nginx, данная команда показывает в реальном времени лог:
```terminal
sudo tail -f /var/log/nginx/access.log
```

Так же есть лог с ошибками, будем использовать вывод только последних 50 строк:
```terminal
sudo tail -n 50 /var/log/nginx/error.log
```

Если не находим причину поломки, то используем [[Нужные команды для DevOps Engineer#Journalctl| journalctl]]

## Продвинутый уровень

Для того чтобы определить в чем проблем приходиться копаться глубжде в логах для этого мы используем journalctl.

Чтобы смотреть только логи nginx используем эту команду:
```terminal
sudo journalctl -u nginx
```

Данная команда показывает конец журнала, помогает понять на какой ошибке остановился сервис(очень выручает при дебаге):
```terminal
sudo journalctl -xe
```

Наблюдать за всей системой в реальном времени:
```terminal
sudo journalctl -f
```
[[Нужные команды для DevOps Engineer#Journalctl| journalctl]]

```termial
cat /var/log/nginx/access.log | grep "192.168.101.131"
```
